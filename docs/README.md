## Navigation convention

Every folder has a `README.md` so GitHub renders a proper index. Breadcrumbs at the top of each doc link to the parent `README.md`. Dated entries in [decisions.md](project/decisions.md) and [worklog.md](process/worklog.md) are chronological — oldest first, newest appended at the bottom. The `CHANGELOG.md` at the repo root keeps its own descending convention (Keep a Changelog standard).
